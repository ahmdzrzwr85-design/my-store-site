# A.H Store

هذا مشروع متجر بسيط لعرض وبيع ملابس وحقائب مع سلة شراء.

## مميزات المشروع

- صفحة عرض المنتجات `index.html`
- صفحة سلة المشتريات `cart.html`
- حفظ السلة محلياً عبر `localStorage`
- دعم طلبات الحفظ عبر `server.js` (Node.js + Express)
  -- خيار دفع نقدي عند الاستلام
- إمكانية حفظ الطلبات على الخادم عبر `/api/orders`

## تشغيل المشروع محلياً

1. افتح الطرفية في المجلد `c:\askan`
2. شغّل:
   - `npm install`
   - `npm start`
3. افتح المتصفح على:
   - `http://localhost:3000/index.html`

## نشر الموقع على الإنترنت (مع حفظ الطلبات)

للحصول على الموقع كاملًا وجاهزًا للبيع مع حفظ الطلبات، أنصح باستضافة Node.js مثل Render أو Railway.

### نشر على Render

1. أنشئ حساب في `https://render.com`
2. أنشئ مستودع جديد على GitHub ودفع الكود إليه
3. اضغط `New Web Service`
4. اربط حساب GitHub واختر المستودع
5. ضع:
   - Build Command: `npm install`
   - Start Command: `npm start`
6. اضغط نشر وسيكون الرابط جاهز

## تفاصيل إضافية

- `orders.json` يستخدم لتخزين الطلبات على الخادم، وهو غير مضمَّن في Git بواسطة `.gitignore`
- إذا أردت نشر الموقع كصفحة ثابتة فقط، استخدم GitHub Pages أو Netlify، لكن حفظ الطلبات لن يعمل في هذه الحالة.

## دمج دفع بالبطاقة (محاكاة)

أضفت صفحة دفع بالبطاقة محلية للتجربة (`card-payment.html`) والتي تسمح بإدخال بيانات البطاقة بشكل محاكٍ — لا تُخزن سوى آخر 4 أرقام من البطاقة.

كيفية التجربة:

1. أضف منتجات إلى السلة واملأ بيانات الطلب في [index.html](index.html) أو [cart.html](cart.html) ثم اختر "بطاقة ائتمان" واضغط "إتمام الطلب".
2. سيتم توجيهك إلى [card-payment.html](card-payment.html) لإدخال بيانات البطاقة (محاكاة).
3. الصفحة سترسل بيانات البطاقة بشكل محمي افتراضياً إلى نقطة النهاية `/api/card-pay` على الخادم، وستحفظ الطلب مع معلومات بطاقة مقنّعة (النوع وآخر 4 أرقام).

ملاحظة أمان: هذا تنفيذ تعليمي/محاكٍ فقط — لا يُستعمل في الإنتاج. لتمكين معاملات بطاقات حقيقية، استخدم مزوّد دفع موثوق (مثل Stripe أو PayTabs) واتبع ممارسات PCI-DSS.

## قاعدة بيانات لحسابات البنوك وسحوبات المدفوعات (محاكاة)

تمت إضافة دعم تخزين حسابات بنكية مُشفّرة وسجل سحوبات (payouts). هذه الميزة تستخدم Postgres وتشفير AES-256-GCM لحماية بيانات الـ IBAN/رقم الحساب.

- ملفات هامة:
  - `db.js` : موصل Postgres ووظائف التشفير/فك التشفير.
  - `db/migrations/init.sql` : مخطط الجداول (`bank_accounts`, `payouts`).
  - endpoints:
    - `POST /api/bank-account` : احفظ حسابًا بنكيًا مشفراً. جسم الطلب: `{ ownerName, bankName, iban }`.
    - `GET /api/bank-account/:id` : استرجاع بيانات الحساب (لا يعيد IBAN المشفّر بشكل افتراضي).
    - `POST /api/payout` : سجّل سحبًا (محاكاة). جسم الطلب: `{ bankAccountId, amountCents, currency }`.

إعداد البيئة:

1. انشئ قاعدة Postgres واربطها عبر `DATABASE_URL`.
2. أنشئ مفتاح تشفير 32-byte بصيغة base64 وعرّفه في `ENCRYPTION_KEY`.
3. مثال على ملف `.env.example` موجود في المشروع.

مهم: التنفيذ الحالي "محاكاة" لسحوبات Visa — للمدفوعات الحقيقية/تحويلات بنكية استخدم مزوّد دفع/بوابة تدعم Payouts (مثل Stripe Connect أو مزود بنكي) واتباع متطلبات PCI وKYC.

تحويل العملات وحساب الشحن بالوزن

- الموقع الآن يعرض الأسعار ويحسب إجمالي الطلب بالدولار الأمريكي. التحويل يتم داخل `script.js` بواسطة المتغير `EGP_TO_USD` ومعدل الشحن `SHIPPING_USD_PER_KG`.
- لتعديل سعر الصرف أو معدل الشحن، افتح `script.js` وابحث عن:

```js
const EGP_TO_USD = 0.032; // 1 EGP -> USD
const SHIPPING_USD_PER_KG = 2.5; // شحن بالدولار لكل كجم
```

غيّر القيم حسب الحاجة ثم أعد تحميل الصفحة.
